# Contributing to AgriConnect

Thank you for your interest in contributing to AgriConnect! We welcome contributions from the community.

## How to Contribute

### 1. Fork the Repository
Click the "Fork" button on GitHub to create your own copy of the project.

### 2. Clone Your Fork
```powershell
git clone https://github.com/YOUR_USERNAME/AgriConnect.git
cd AgriConnect
```

### 3. Create a Feature Branch
```powershell
git checkout -b feature/your-feature-name
```

Use descriptive branch names:
- `feature/add-marketplace-item` for new features
- `bugfix/fix-login-issue` for bug fixes
- `docs/update-readme` for documentation updates

### 4. Make Your Changes
- Write clean, well-documented code
- Follow the existing code style
- Add comments for complex logic

### 5. Commit Your Changes
```powershell
git add .
git commit -m "Brief description of changes"
```

### 6. Push to Your Fork
```powershell
git push origin feature/your-feature-name
```

### 7. Create a Pull Request
1. Go to the original repository on GitHub
2. Click "New Pull Request"
3. Select your branch
4. Add a descriptive title and description
5. Submit the PR

## Code Style Guidelines

- **JavaScript/React:** Use ES6+ syntax
- **File Names:** Use lowercase with hyphens (e.g., `crop-recommendation.js`)

## Testing

Before submitting a PR:
```powershell
# Frontend/Backend tests
npm test
```

## Reporting Issues

Found a bug? Please create an issue with:
- Clear title and description
- Steps to reproduce
- Expected vs actual behavior
- Screenshots (if applicable)

## Questions?

Open a discussion or create an issue with the label "question".
